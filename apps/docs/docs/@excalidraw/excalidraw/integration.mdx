# Integration

## Module bundler

If you are using a module bundler (for instance, Webpack), you can import it as an ES6 module as shown below

```js
import { Drawink } from "@drawink/drawink";
```

:::info

Throughout the documentation we use live, editable Drawink examples like the one shown below.

While we aim for the examples to closely reflect what you'd get if you rendered it yourself, we actually initialize it with some props behind the scenes.
For example, we're passing a `theme` prop to it based on the current color theme of the docs you're just reading.

:::

```jsx live
function App() {
  return (
    <>
      <h1 style={{ textAlign: "center" }}>Drawink Example</h1>
      <div style={{ height: "500px" }}>
        <Drawink />
      </div>
    </>
  );
}
```

### Next.js

Since Drawink doesn't support `server side rendering` so it should be rendered only on `client`. The way to achieve this in next.js is using `next.js dynamic import`.

If you want to only import `Drawink` component you can do :point_down:

```jsx showLineNumbers
import dynamic from "next/dynamic";
import "@drawink/drawink/index.css";

const Drawink = dynamic(
  async () => (await import("@drawink/drawink")).Drawink,
  {
    ssr: false,
  },
);
export default function App() {
  return <Drawink />;
}
```

However the above component only works for named component exports. If you want to import some util / constant or something else apart from Drawink, then this approach will not work. Instead you can write a wrapper over Drawink and import the wrapper dynamically.

If you are using `pages router` then importing the wrapper dynamically would work, where as if you are using `app router` then you will have to also add `useClient` directive on top of the file in addition to dynamically importing the wrapper as shown :point_down:

### Preact

Since we support `umd` build ships with `react/jsx-runtime` and `react-dom/client` inlined with the package. This conflicts with `Preact` and hence the build doesn't work directly with `Preact`.

However we have shipped a separate build for `Preact` so if you are using `Preact` you need to set `process.env.IS_PREACT` to `true` to use the `Preact` build.

Once the above `env` variable is set, you will be able to use the package in `Preact` as well.

:::info

When using `vite` or any build tools, you will have to make sure the `process` is accessible as we are accessing `process.env.IS_PREACT` to decide whether to use `preact` build.

Since Vite removes env variables by default, you can update the vite config to ensure its available :point_down:

```
 define: {
    "process.env.IS_PREACT": JSON.stringify("true"),
  },
```
:::

## Browser

To use it Drawink in a browser directly, use the following setup :point_down:

> **Note**: We rely on import maps to de-duplicate `react`, `react-dom` and `react/jsx-runtime` versions.

import Tabs from "@theme/Tabs";
import TabItem from "@theme/TabItem";

<Tabs>
  <TabItem value="html" label="html">

```html
<!DOCTYPE html>
<html>
  <head>
    <title>Drawink in browser</title>
    <meta charset="UTF-8" />
    <link
      rel="stylesheet"
      href="https://esm.sh/@drawink/drawink@0.18.0/dist/dev/index.css"
    />

    <script>
    	window.EXCALIDRAW_ASSET_PATH = "https://esm.sh/@drawink/drawink@0.18.0/dist/prod/";
		</script>
    <script type="importmap">
```

```js showLineNumbers
// See https://www.npmjs.com/package/@drawink/drawink documentation.
import * as DrawinkLib from 'https://esm.sh/@drawink/drawink@0.18.0/dist/dev/index.js?external=react,react-dom';
import React from "https://esm.sh/react@19.0.0";
import ReactDOM from "https://esm.sh/react-dom@19.0.0"

window.DrawinkLib = DrawinkLib;
console.log("Drawink library", DrawinkLib);

const App = () => {
  return React.createElement(
    React.Fragment,
    null,
    React.createElement(
      "div",
      {
        style: { height: "500px" },
      },
      React.createElement(DrawinkLib.Drawink),
    ),
  );
};

const drawinkWrapper = document.getElementById("app");
const root = ReactDOM.createRoot(drawinkWrapper);
root.render(React.createElement(App));
```
